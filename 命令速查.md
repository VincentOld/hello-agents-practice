## 1. Git —— 仓库与提交循环

| 命令 | 作用 |
| --- | --- |
| `git init -b main` | 在当前目录新建一个 git 仓库,默认分支叫 `main` |
| `git status` | 查看当前状态:在哪个分支、哪些文件改了/没跟踪/待提交（最常用） |
| `git add 文件` | 把改动放进暂存区(打包篮);`git add .` = 当前目录所有改动 |
| `git commit -m "说明"` | 把暂存区的内容打包成一个版本快照,`-m` 后面是提交说明 |
| `git log` | 查看提交历史(详细);按 `q` 退出 |
| `git log --oneline` | 查看历史(每个提交一行,清爽) |
| `git diff` | 看「还没暂存」的改动;`git diff 文件` 只看某个文件 |
| `git config core.quotepath false` | 让中文文件名正常显示(否则是 `\346\257...` 转义) |
| `git check-ignore -v 文件` | 查某文件是否被 `.gitignore` 忽略、被哪条规则忽略 |

**`.gitignore`**:一个文本文件,每行一条「不让 git 管」的规则。常用写法:
- `__pycache__/` 末尾 `/` = 匹配文件夹
- `*.pyc` `*` = 通配符(任意字符)
- `.env` = 精确匹配名为 `.env` 的文件(密钥文件,绝不提交!)
- `#` 开头 = 注释

## 2. Git —— 分支与 worktree

| 命令 | 作用 |
| --- | --- |
| `git branch` | **列出**所有分支(`*` 是当前分支;`+` 表示被其他 worktree 占用) |
| `git branch 名字` | ⚠️ **创建**一个叫「名字」的新分支(不是列出!) |
| `git branch -m 旧名 新名` | 给分支改名(`-m` = move/rename) |
| `git branch -d 名字` | **安全删除**分支(只有已合并才允许删);`-D` 强制删 |
| `git switch 分支` | 切换到某分支;`git switch -c 新分支` = 创建并切换 |
| `git merge 分支` | 把「分支」合并进当前分支(常见 Fast-forward 快进合并) |

**worktree(工作树)**:让同一个仓库的不同分支同时签出到不同文件夹,互不干扰。完整流程:

```bash
git worktree add -b 功能分支 路径    # ① 建功能分支 + 独立工作树
# ② 进那个文件夹改代码 → git add → git commit（提交落在功能分支，main 不动）
git merge 功能分支                    # ③ 回 main 工作树执行，把功能并进来
git worktree remove 路径             # ④ 删掉临时工作树
git branch -d 功能分支               #    删掉已合并的分支
git worktree list                    # 随时查看所有工作树
```

## 3. Git —— 远程仓库(GitHub)

| 命令 | 作用 |
| --- | --- |
| `gh repo create 名字 --public` | 用 GitHub 命令行在云端新建公开仓库(`--private` 则私有) |
| `gh auth status` | 查看 GitHub 登录状态 |
| `git remote add origin 地址` | 把本地仓库连到远程,起别名 `origin` |
| `git remote -v` | 查看已配置的远程(`-v` 带出地址) |
| `git remote remove origin` | 删除远程连接 |
| `git push -u origin main` | 首次推送:把本地 `main` 推到远程,并**绑定追踪**(`-u`) |
| `git push` | 日常推送:绑定过追踪后,光这一个词就够了 |
| `git pull` | 把远程的新提交拉到本地 |

> `origin/main` 是「远程追踪分支」——git 本地记着的「远程上次到哪了」的快照。本地领先它,就说明有提交还没 push。

## 4. Python 虚拟环境(venv)

| 命令 | 作用 |
| --- | --- |
| `py -3.10 --version` | 用 Windows 的 py 启动器,指定调用 Python 3.10 |
| `py -3.10 -m venv .venv` | 用 3.10 在 `.venv\` 里创建隔离环境 |
| `.\.venv\Scripts\Activate.ps1` | 激活环境(PowerShell);激活后行首出现 `(.venv)` |
| `python --version` / `pip --version` | 验证当前用的是哪个 python/pip |
| `python -m pip install --upgrade pip` | 升级 pip(`python -m pip` 比直接 `pip` 更稳) |
| `pip install 包名` | 安装包;`pip install -r requirements.txt` 按清单装 |
| `Set-ExecutionPolicy -Scope CurrentUser RemoteSigned` | 解决 PowerShell「禁止运行脚本」导致激活失败 |

> 多个 Python 版本共存时,直接敲 `python` 有歧义(指向 PATH 第一个)。用 `py -3.10` 精确指定。

## 7. 踩过的坑(重点记忆)

- **`git branch list` 不是列出分支**,而是建了个叫 `list` 的分支!列出用不带参数的 `git branch`。
- **Streamlit 用 `python xxx.py` 跑没反应**(只刷一堆 `missing ScriptRunContext` 警告)——必须 `streamlit run xxx.py`。
- **`st.experimental_rerun()` 已被删除**,新版 Streamlit 用 `st.rerun()`。
- **非 OpenAI 模型(如 DeepSeek)配 AutoGen**,`OpenAIChatCompletionClient` 必须传 `model_info` 声明能力,否则报错。
- **`git push` 报 `Connection was reset`**:常是网络/GFW 对 `github.com` 的干扰;在自己挂了代理的终端里推、或给 git 配代理可解决。
- **`requirements.txt` 别照抄**:`asyncio` 是标准库不用装;`dotenv` 的正确包名是 `python-dotenv`。
